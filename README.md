# Excel転記くん（OCR読み取り精度強化版）

Excel転記くんは、写真やスキャン画像に写っている文字を読み取り、食材名・数量・単位を抽出してExcelにまとめるアプリです。

現在は **ファイル読み取り → 自動抽出 → Excel反映** に集中しています。手入力だけでExcelを作る機能は停止しています。

## 何をするアプリ？

- スマホ写真やスキャン画像を読み取ります。
- 読み取った全文を画面に表示します。
- OCR全文から、曜日付きの「3歳未満児」食材名・基準量・単位だけを探します。
- 月曜日は5人分、火曜日から金曜日は7人分で自動計算します。
- 抽出できた場合だけ、結果を `output/ocr_転記結果.xlsx` に保存します。
- 読み取りがあやしい行は「要確認」にして、Excelで色を付けます。
- 元の画像名と、補正した画像の保存場所もExcelに残します。
- OCR結果・抽出結果・失敗理由を画面とログに残します。
- OCR結果や抽出結果が空の場合、空欄Excelは作りません。

## 画像はどこに入れる？

読み取りたい画像を、次のフォルダに入れてください。

```bash
input_images/
```

使える画像は次の形式です。

- jpg
- jpeg
- png
- bmp
- tiff / tif

画像PDFは今後対応予定です。

## 最初の準備

Pythonを使います。まず、必要なものを入れてください。

```bash
pip install -r requirements.txt
```

Tesseract OCR本体も必要です。パソコンに入っていない場合は、先にTesseract OCRをインストールしてください。  
日本語を読む場合は、日本語データも入れてください。

## 実行方法

画像を `input_images/` に入れたら、次のコマンドを実行します。

```bash
python run_ocr.py
```

終わると、次のExcelができます。

```bash
output/ocr_転記結果.xlsx
```

## Excelの見方

Excelには次の列が出ます。

| 列 | 内容 |
| --- | --- |
| A | 処理日時 |
| B | 元ファイル名 |
| C | 採用した向き |
| D | 補正方法 |
| E | OCRエンジン |
| F | 読み取り全文 |
| G | 抽出した日付 |
| H | 抽出した金額 |
| I | 抽出した数量 |
| J | 抽出した電話番号 |
| K | 信頼度 |
| L | 要確認フラグ |
| M | 備考 |
| N | 補正後画像パス |

### 色の意味

- 薄い黄色：読み取り結果を確認してください。
- 薄い赤：信頼度がかなり低いです。必ず元画像と見比べてください。

数字、金額、日付は間違えると困るため、信頼度が低い場合は必ず「要確認」になります。

## 補正後画像の保存場所

OCR前に補正した画像は、ここに保存されます。

```bash
output/processed_images/
```

ExcelのN列にも保存場所が書かれます。

## 読み取りが悪い時の写真の撮り方

- できるだけ明るい場所で撮ってください。
- 紙全体が入るように撮ってください。
- 文字がぼやけないように、少し待ってから撮ってください。
- 影が入らないようにしてください。
- 斜めからではなく、できるだけ正面から撮ってください。
- 小さい文字は、少し近づいて大きく撮ってください。
- くしゃくしゃの紙は、できるだけ平らにしてください。

## エラー時の確認方法

うまく読めない画像があっても、処理は止まりません。  
理由は次のログに残ります。

```bash
output/ocr_log.txt
```

ExcelのM列「備考」にも、エラーや要確認の理由が出ます。

## 手入力機能について

手入力だけでExcelを作る機能は停止しています。

1. `index.html` をブラウザで開きます。
2. ファイルを選びます。
3. 読み取った全文が画面に表示されます。
4. 食材名・数量・単位を抽出できた時だけ **Excel出力** を押せます。

## GitHub Pages の設定

このリポジトリは、ビルド処理を使わずに `index.html` だけを GitHub Pages で公開する静的サイトです。

1. GitHubでこのリポジトリを開きます。
2. 上のメニューから **Settings** を押します。
3. 左のメニューから **Pages** を押します。
4. **Build and deployment** の **Source** を **Deploy from a branch** にします。
5. **Branch** は公開したいブランチ、フォルダは **/ (root)** を選びます。
6. **Save** を押します。
7. 表示された公開URLをスマホやPCのブラウザで開くと `index.html` が表示されます。

## ファイル構成

- `index.html`: ブラウザ版です。ファイル読み込み、3歳未満児量の人数計算、単位変換、発注書Excel出力を行います。
- `run_ocr.py`: 追加したOCR強化版です。`input_images/` の画像を読み取り、3歳未満児量を人数計算してExcelへ転記します。
- `requirements.txt`: OCR強化版に必要なPythonライブラリです。
- `input_images/`: 読み取りたい画像を入れる場所です。
- `output/`: Excel、補正後画像、ログが保存される場所です。

## Streamlit版 MVP

画像/PDFをアップロードし、OCR全文表示、食材候補の確認・修正、Excel出力までを行う最小版です。

### 実行方法

```bash
pip install -r requirements.txt
streamlit run app.py
```

### MVPでできること

- jpg / jpeg / png / pdf のアップロード
- PaddleOCR（入っている場合）またはTesseract OCRによる全文表示
- OCR信頼度の表示
- 数字と単位を含む行から食材候補を抽出
- `data/food_master.csv` による食材名の表記ゆれ補正
- 画面上で食材候補を確認・修正
- 新しいExcelファイルとしてダウンロード

### 安全ルール

- OCR結果をそのまま発注に使わず、必ず確認・修正画面を通します。
- MVPでは既存Excelを上書きしません。
- 既存Excelフォーマットへの転記は、実際の発注書ファイルを確認してから対応します。
