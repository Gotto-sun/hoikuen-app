# ほいくえんアプリ

このリポジトリは、ビルド処理を使わずに `index.html` だけを GitHub Pages で公開する静的サイトです。

## GitHub Pages の設定

GitHub Actions や Jekyll は使いません。GitHub Pages はブランチ直下の静的ファイルをそのまま配信する設定にしてください。

1. GitHubでこのリポジトリを開きます。
2. 上のメニューから **Settings** を押します。
3. 左のメニューから **Pages** を押します。
4. **Build and deployment** の **Source** を **Deploy from a branch** にします。
5. **Branch** は公開したいブランチ（例: `work`）、フォルダは **/ (root)** を選びます。
6. **Save** を押します。
7. 表示された公開URLをスマホやPCのブラウザで開くと `index.html` が表示されます。

## 構成

- `index.html`: アプリ本体です。HTML、CSS、JavaScript をこの1ファイル内にまとめています。献立CSVまたは手入力テーブルから「材料名・数量・単位」を読み取り、コード内標準ルールによる発注単位変換、A4縦サイズの発注書兼納品書Excel作成までブラウザ内で実行します。画像OCRからの自動解析は行いません。
- `.nojekyll`: GitHub Pages に Jekyll 処理をさせないための空ファイルです。

## 使い方

1. スマホまたはPCのブラウザで `index.html` を開きます。
2. **献立CSVを読み込む** から、1行目に「材料名,数量,単位」があるCSV/TXT/TSVを選びます。
3. CSVファイルがない場合は、**手入力テーブル** に `材料名,数量,単位` の形式で貼り付け、**手入力テーブルを反映** を押します。
4. 発注書の列（チェック、材料名、総使用量、単位、摘要、納品数、単位、単価、金額、産地）を確認します。
5. **Excel出力** を押してください。A4縦1ページに収まる印刷設定付きの発注書兼納品書Excelを出力します。
6. 必要に応じて **A4印刷** を押し、ブラウザから印刷します。

### 献立CSVの形式

```csv
材料名,数量,単位
にんじん,120,g
玉ねぎ,0.5,kg
卵,3,個
```

※画像OCRから無理に解析しないため、材料名・数量・単位の3列で入力してください。

## ページを書きかえる方法

`index.html` を編集して保存し、公開ブランチに push してください。GitHub Pages がブランチ直下の `index.html` をそのまま配信します。
