# ほいくえんアプリ

このリポジトリは、ビルド処理を使わずに `index.html` だけを GitHub Pages で公開する静的サイトです。

## GitHub Pages の設定

GitHub Actions や Jekyll は使いません。GitHub Pages はブランチ直下の静的ファイルをそのまま配信する設定にしてください。

1. GitHubでこのリポジトリを開きます。
2. 上のメニューから **Settings** を押します。
3. 左のメニューから **Pages** を押します。
4. **Build and deployment** の **Source** を **Deploy from a branch** にします。
5. **Branch** は公開したいブランチ（例: `work`）、フォルダは **/ (root)** を選びます。
6. **Save** を押します。
7. 表示された公開URLをスマホやPCのブラウザで開くと `index.html` が表示されます。

## 構成

- `index.html`: アプリ本体です。HTML、CSS、JavaScript をこの1ファイル内にまとめています。献立CSV/OCR結果の解析、園児数CSVの反映、発注ルールCSVまたはコード内標準ルールによる発注単位変換、発注書兼納品書xlsx/CSVの作成までブラウザ内で実行します。
- `.nojekyll`: GitHub Pages に Jekyll 処理をさせないための空ファイルです。

## 使い方

1. スマホまたはPCのブラウザで `index.html` を開きます。
2. **献立CSVを選ぶ**、または **献立写真を撮る / 画像を選ぶ** から献立データを取り込みます。
3. 必要に応じて **園児数CSVを選ぶ** から3歳未満児の人数を読み込みます。未選択の場合は1人分として計算します。
4. 必要に応じて **発注ルールCSVを選ぶ** から園独自の発注単位を読み込みます。未選択の場合はコード内の標準ルールを使います。
5. **発注数を計算する** を押し、発注書の列（チェック、材料名、総使用量、単位、摘要、納品数、単位、単価、金額、産地）を確認します。
6. 必要に応じて納品日・使用日・検品者を入力します。
7. 手直し後に **発注書Excel（.xlsx）をダウンロード** を押してください。
8. スマホでExcel保存がうまくいかない場合は、**CSVをダウンロード（スマホ・Googleスプレッドシート用）** を押してください。UTF-8 BOM付きCSVなので、Googleスプレッドシートでも文字化けしにくい形式です。

## ページを書きかえる方法

`index.html` を編集して保存し、公開ブランチに push してください。GitHub Pages がブランチ直下の `index.html` をそのまま配信します。
